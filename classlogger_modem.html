Construct a new logger Modem object.

Parameters
powerPinThe digital pin number of the pin suppling power to the modem (active HIGH)
statusPinThe digital pin number of a pin indicating modem status
statusLevelThe level of the status pin (0/LOW or 1/HIGH) when the modem is active
modemResetPinThe digital pin number of a pin for hard or panic resets
resetLevelThe level on the modemResetPin which will reset the modem
resetPulse_msThe length of time in ms required to reset the modem using a panic reset
modemSleepRqPinThe digital pin number of a pin used to request the modem enter its lowest possible power state
wakeLevelThe level the modemSleepRqPin should be set to to wake the modem
wakePulse_msThe length of time the modemSleepRqPin should be held at the wakeLevel in order to wake the modem. Set to 0 if the pin must be continuously held to the level to keep the modem at fully functional state.
max_status_time_msThe maximum length of time in milliseconds between when power is supplied to the modem and when the statusPin should reach the statusLevel.
max_disconnetTime_msThe maximum length of time in milliseconds between when the modem is requested to enter lowest power state and when it should have completed necessary steps to shut down.
wakeDelayTime_msThe time in milliseconds between when power is supplied to the modem and when the wake function can be used to request the modem enter fully power operation.
max_atresponse_time_msThe maximum length of time in milliseconds between when the modem has been requested to wake and when it begins responding to commands over the main serial connection.

virtual bool loggerModem::connectInternet ( uint32_t  maxConnectionTime = 50000L)
pure virtual

Waits for the modem to successfully register on the cellular network and then requests that it establish either EPS or GPRS data connection.

Parameters
maxConnectionTimeThe maximum length of time in milliseconds to wait for network registration and data sconnection. Defaults to 50,000ms (50s).
Returns
true EPS or GPRS data connection has been established
false The modem was either unable to communicated, unable to register with the cellular network, or unable to establish a EPS or GPRS connection.

◆ modemHardReset()

bool loggerModem::modemHardReset ( void  )
virtual

Uses the modem reset pin specified in the constructor to signal the modem for a "hard" or "panic" reset.

This should only be used if the modem is clearly non-responsive.

Returns
true The reset succeeded and the modem should now be responsive
false The modem remains non-responsive either because the reset failed to fix the communication issue or because a reset is not possible with the current pin/modem configuration.

Reimplemented in SodaqUBeeR410M, QuectelBG96, DigiXBee3GBypass, and DigiXBeeLTEBypass.
